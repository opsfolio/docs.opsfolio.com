## Installation

Spry can be installed via multiple methods depending on your platform and preferences. Choose the method that works best for your workflow.

<Tabs items={['Homebrew', 'Ubuntu', 'Debian', 'macOS', 'Windows']}>
  <Tab value="Homebrew">
    ### Homebrew (Recommended for macOS & Linux)
    
    The easiest way to install Spry on macOS or Linux is via Homebrew:
    ```bash
        # Install via Homebrew
        brew install programmablemd/packages/spry
    ```
    Or add the tap first, then install:
    ```bash
        brew tap programmablemd/homebrew-packages
        brew install spry
    ```
    <Callout type="info">
    Homebrew automatically manages updates and dependencies, making it the recommended installation method for most users.
    </Callout>
    #### Install Specific Version
    To install a specific version of Spry:
    **Option 1: Using version suffix**
    ```bash
        brew install programmablemd/packages/spry@1.0.2
    ```
    **Option 2: Switch between installed versions**
    If you have multiple versions installed:
    ```bash
        brew unlink spry@1.0.2 && brew link spry@1.0.3
    ```
  </Tab>

  <Tab value="macOS">
    ### macOS Binary Download
    
    Download pre-built binaries from the [GitHub Releases page](https://github.com/programmablemd/packages/releases).
    
    #### Installation Steps:
    ```bash
    wget https://github.com/programmablemd/packages/releases/download/latest/spry-macos.tar.gz
    tar -xzf spry-macos.tar.gz
    sudo cp spry-macos /usr/local/bin/spry
    chmod +x /usr/local/bin/spry
    ```
    
    #### Versioned Packages (v1.0.3) for macOS
    ```bash
        # Download and extract the archive
        wget https://github.com/programmablemd/packages/releases/download/v0.80.0/spry-sqlpage-macos.tar.gz
        tar -xzf spry-sqlpage-macos.tar.gz
        
        # Move to your PATH
        sudo mv spry-sqlpage-macos /usr/local/bin/spry-sqlpage
    ```
    
    <Callout type="info">
    Binaries support both Intel and Apple Silicon Macs.
    </Callout>
    
    <Callout type="warning">
    Manual binary installations won't receive automatic updates. You'll need to download new versions manually.
    </Callout>
  </Tab>
  
  <Tab value="Windows">

    ### Download Latest Stable Version for Windows
     Download the [spry-windows.zip](https://github.com/programmablemd/packages/releases/download/latest/spry-windows.zip).

    ### Download Versioned Packages (v1.0.3)- Installation Steps :
     Download the `.zip` file for your desired version from the [GitHub Releases page](https://github.com/programmablemd/packages/releases/)

    ### Installation Steps :
    1. Extract the `.zip` file to a location of your choice
    2. Add the extracted directory to your system PATH:
       - Open System Properties → Advanced → Environment Variables
       - Edit the `Path` variable and add the directory containing `spry.exe`
    3. Restart your terminal or command prompt
  
    
    #### Verify Installation:

    ```bash
        spry --version
        spry --help
    ```
    
    <Callout type="warning">
    Manual binary installations won't receive automatic updates. You'll need to download new versions manually.
    </Callout>

### Verify Installation

<Steps>
  <Step>
    ### Check Version
    
    Confirm Spry is installed correctly by checking the version:

    ```bash
    spry --version
    ```

    You should see output like:
    ```
    Spry vx.x.x
    #Example:
    Spry v1.0.3
    ```
  </Step>

  <Step>
    ### View Available Commands
    
    Display the help menu to see all available commands:

    ```bash
    spry --help
    ```

    This shows the main command groups:
    - `axiom` - Document structure exploration
    - `rb` (runbook) - Task execution
    - `sp` (sqlpage) - SQLPage application management
  </Step>

  <Step>
    ### Test Execution
    
    Create a simple test file to verify execution works:

    ```bash
    echo '```bash test
    echo "Hello from Spry!"
    ```' > test.md
    
    spry rb run test.md
    ```

    You should see:
    ```
    Hello from Spry!
    ```
  </Step>
</Steps>

### System Requirements

#### Minimum Requirements

- **Operating System**: macOS 10.15+, Ubuntu 20.04+, Windows 10+, or equivalent Linux distribution
- **RAM**: 512 MB minimum, 2 GB recommended
- **Disk Space**: 50 MB for Spry binary, plus space for your projects

#### Optional Dependencies

Some features require additional tools:

| Tool | Purpose | Installation |
|------|---------|--------------|
| **SQLPage** | Run SQLPage web applications | [sqlpage.com](https://sqlpage.com) |
| **direnv** | Automatic environment variable loading | `brew install direnv` or [direnv.net](https://direnv.net) |
| **Git** | Version control integration | Pre-installed on most systems |

<Callout type="info">
**SQLPage for Web Applications**

If you plan to build SQLPage applications, install SQLPage separately:

```bash
# macOS/Linux with Homebrew
brew install sqlpage

# Or download from https://github.com/lovasoa/SQLpage/releases
```
</Callout>

### Environment Setup

#### Using direnv (Recommended)

Spry works well with `direnv` for automatic environment variable management:

<Steps>
  <Step>
    ### Install direnv
    
    ```bash
    # macOS
    brew install direnv
    
    # Ubuntu/Debian
    sudo apt-get install direnv
    ```
  </Step>

  <Step>
    ### Configure Your Shell
    
    Add to your shell configuration:

    ```bash
    # For bash (~/.bashrc)
    eval "$(direnv hook bash)"
    
    # For zsh (~/.zshrc)
    eval "$(direnv hook zsh)"
    ```
  </Step>

  <Step>
    ### Create .envrc File
    
    Spry can generate `.envrc` files automatically:

    ```markdown
    ```envrc prepare-env -C ./.envrc --gitignore --descr "Generate .envrc file and add it to local .gitignore if it's not already there"
    export SPRY_DB="sqlite://app.db?mode=rwc"
    export PORT=9227
    ```
    ```

    Then allow direnv to load it:
    ```bash
    direnv allow
    ```
  </Step>
</Steps>

#### Manual Environment Setup

If you prefer not to use direnv, you can source environment files manually:

```bash
# Create environment file
cat > .envrc << 'EOF'
export SPRY_DB="sqlite://app.db?mode=rwc"
export PORT=9227
EOF

# Load environment
source .envrc
```
